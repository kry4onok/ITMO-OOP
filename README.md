# ITMO OOP

Репозиторий с лабораторными работами по дисциплине "Объектно-ориентированное
программирование" в ИТМО.

Решение написано на C# и .NET 9. Внутри пять независимых лабораторных работ с
unit- и functional-тестами. Основной фокус: объектная модель, паттерны
проектирования, слоистая архитектура и тестируемая бизнес-логика.

## Краткое описание

**ITMO Object-Oriented Programming Labs** - набор из пяти учебных проектов на
C#/.NET 9: симулятор маршрута поезда, система рассылки сообщений, пошаговая
боевая модель существ, консольный файловый менеджер и банковский API.

В репозитории показаны принципы ООП, паттерны проектирования, работа с
абстракциями, слоистая архитектура, тестирование через xUnit и строгая проверка
кода analyzers. Проект демонстрирует не только банковскую систему из Lab 5, а
полный набор лабораторных с разными доменными моделями и архитектурными
подходами.

## Стек

- C# и .NET 9
- xUnit для тестов
- NSubstitute и Moq для тестовых double-объектов
- StyleCop и SourceKit analyzers
- Централизованное управление версиями пакетов через `Directory.Packages.props`
- ASP.NET Core, Swagger, Mediator и Microsoft DI в Lab 5

## Структура

```text
src/
  Lab1/                 Доменная модель маршрута поезда
  Lab2/                 Доменная модель рассылки сообщений

  Lab4.Core/            Ядро файлового менеджера
  Lab4.Presentation/    Консольная точка входа файлового менеджера

tests/
  Lab1.Tests/
  Lab2.Tests/
  Lab3.Tests/
  Lab4.Tests/
  Lab5.Tests/
```

## Запуск

Восстановить зависимости, собрать проекты и запустить все тесты:

```bash
dotnet test
```

Запустить консольное приложение Lab 4:

```bash
dotnet run --project src/Lab4.Presentation/Lab4.Presentation.csproj
```

Запустить API Lab 5:

```bash
dotnet run --project src/Lab5/Presentation/Presentation.csproj
```

Lab 5 использует in-memory репозитории, поэтому счета и сессии живут только пока
запущено приложение. Демонстрационный пароль администратора лежит в
`src/Lab5/Presentation/appsettings.json`.

# Лабораторная 2: система рассылки сообщений

Лаба реализует модель доставки сообщений пользователям и другим адресатам. В
системе есть темы, адресаты, фильтрация по важности, логирование, проверка
уведомлений и архивирование.

## Идея

`Topic` отправляет сообщение одному или нескольким адресатам. Адресатом может
быть пользователь, группа, архив или система уведомлений. Дополнительное
поведение можно навешивать на адресатов через обертки, не меняя основную
реализацию.

У сообщения есть заголовок, тело и уровень важности. Пользователь хранит
полученные сообщения и может отмечать их прочитанными.

## Что использовалось

- Интерфейсы для сообщений и целей доставки.
- Composite-style группировка через `AddresseeGroup`.
- Decorator для логирования через `AddresseeLogging`.
- Proxy/filter-поведение через `AddresseeFilter`.
- Strategy-style интерфейсы форматтера и writer для архивирования.
- xUnit и NSubstitute для проверки поведения.

## Основные файлы

- `Messages/Message.cs` - данные сообщения.
- `Messages/ImportanceLevel.cs` - уровни важности low, normal, high.
- `Clients/User.cs` - inbox пользователя и состояние read/unread.
- `Topics/Topic.cs` - отправка сообщений адресатам.
- `Addressees/AddresseeUser.cs` - адаптер доставки пользователю.
- `Addressees/AddresseeGroup.cs` - групповая доставка.
- `Addressees/ProxyProtection/AddresseeFilter.cs` - фильтр по важности.
- `Addressees/Decorator/AddresseeLogging.cs` - обертка с логированием.
- `Archivers/FormattingArchiver.cs` - форматирование и запись в архив.

## Тесты

Тесты лежат в `tests/Lab2.Tests`.

Проверяются:

- получение непрочитанного сообщения;
- отметка сообщения как прочитанного;
- повторная отметка уже прочитанного сообщения;
- фильтрация сообщений по важности;
- логирование доставки через decorator;
- форматирование и архивирование;
- доставка через topic к фильтрованным и обычным адресатам.

Запуск только тестов Lab 2:

```bash
dotnet test tests/Lab2.Tests/Lab2.Tests.csproj
```

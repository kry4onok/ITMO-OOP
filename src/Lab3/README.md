# Лабораторная 3: боевая модель существ

Лаба моделирует пошаговый бой между двумя столами игроков. На столах находятся
фэнтезийные существа с атакой, здоровьем, особыми правилами поведения,
модификаторами и заклинаниями.

## Идея

У каждого игрока есть `Table` максимум на семь существ. Во время боя игроки по
очереди выбирают живых атакующих и защитников. Бой заканчивается, когда у одного
из игроков не осталось защитников, оба игрока не могут атаковать, либо одна
сторона побеждает.

Существа создаются фабриками, собираются через builder, оборачиваются
модификаторами и изменяются заклинаниями. При добавлении на стол существа
клонируются, чтобы состояние боя не портило исходные объекты.

## Что использовалось

- Factory Method для создания существ.
- Builder для сборки существ с модификаторами.
- Decorator для модификаторов `AttackSkill` и `MagicShield`.
- Prototype-like клонирование через `ICreature.Clone`.
- Полиморфизм для уникального поведения существ.
- Result records для результатов боя.
- xUnit и NSubstitute для unit-, integration- и functional-тестов.

## Доменные объекты

Существа:

- `EvilFighter` удваивает атаку после пережитого урона.
- `CombatAnalyst` увеличивает атаку перед ударом.
- `MimicChest` копирует лучшие значения атаки и здоровья у цели.
- `InfiniteHorror` один раз переживает смертельный урон.
- `MasterAmulet` создается с базовыми характеристиками, а его фабрика добавляет
  модификаторы.

Заклинания и модификаторы:

- `PowerPotion` увеличивает атаку.
- `EndurancePotion` увеличивает здоровье.
- `MagicMirror` меняет местами атаку и здоровье.
- `AmuletProtection` добавляет магический щит.
- `AttackSkill` позволяет атаковать дополнительно при подходящих условиях.
- `MagicShield` блокирует первый входящий урон.

## Основные файлы

- `Creatures/` - базовые контракты и поведение существ.
- `SpecificCreatures/` - конкретные реализации существ.
- `FactoryMethod/` - фабрики существ.
- `Builder/` - fluent-сборка существ.
- `Modifiers/` - decorator-модификаторы и их применение.
- `Spells/` - абстракции и реализации заклинаний.
- `PlayersTables/Table.cs` - стол игрока и коллекция существ.
- `FIght/Battle.cs` - цикл боя и расчет результата.

## Тесты

Тесты лежат в `tests/Lab3.Tests`.

Проверяются:

- поведение существ;
- создание через фабрики;
- применение модификаторов;
- эффекты заклинаний;
- комбинации модификаторов;
- полноценные исходы боя.

Запуск только тестов Lab 3:

```bash
dotnet test tests/Lab3.Tests/Lab3.Tests.csproj
```

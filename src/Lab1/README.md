# Лабораторная 1: симулятор маршрута поезда

Лаба моделирует движение поезда по маршруту из разных типов участков. Нужно
понять, сможет ли поезд пройти весь маршрут, и посчитать время прохождения с
учетом силы, ускорения и ограничений скорости.

## Идея

У поезда есть масса, текущая скорость, текущее ускорение и максимально
допустимая сила. Участки маршрута влияют на поезд по-разному:

- `PowerPath` прикладывает силу и ускоряет или замедляет поезд.
- `CommonPath` пропускает поезд без дополнительной силы.
- `Station` проверяет ограничение скорости на станции.
- `Route` собирает участки в маршрут и проверяет итоговое ограничение скорости.

Расчет движения идет с шагом `accuracy`. Если поезд не может начать или
продолжить движение, сила превышает допустимую, либо финальная скорость выше
ограничения маршрута, прохождение завершается неуспешно.

## Что использовалось

- Инкапсуляция состояния поезда.
- Наследование и полиморфизм для разных типов участков.
- Result objects для успешного и неуспешного прохождения участка/маршрута.
- `decimal` для детерминированных расчетов.
- xUnit-тесты для сценариев маршрута и граничных случаев.

## Основные файлы

- `MyTrain/Train.cs` - состояние поезда и расчет движения.
- `MyRoute/Route.cs` - прохождение маршрута и финальная проверка.
- `MyRoute/RouteSegment.cs` - базовая абстракция участка.
- `MyRoute/PowerPath.cs` - участок с приложенной силой.
- `MyRoute/CommonPath.cs` - обычный участок.
- `MyRoute/Station.cs` - участок станции с ограничением скорости.

## Тесты

Тесты лежат в `tests/Lab1.Tests`.

Проверяются:

- успешный разгон и прохождение маршрута;
- ошибка из-за финального ограничения скорости;
- ограничение скорости на станции;
- маршруты с разгоном и торможением;
- невозможность пройти маршрут без начального движения.

Запуск только тестов Lab 1:

```bash
dotnet test tests/Lab1.Tests/Lab1.Tests.csproj
```
